# Documentation Index

This folder contains the main documentation set for Presage.

## Core Documents

- `project_foundation.md`
  Use this for project purpose, problem statement, solution, required technologies, and technology choices.

- `project_architecture_and_objectives.md`
  Use this for explaining how the project was created, its objectives, architecture, workflow, and intended users.

- `project_security_brief.md`
  Use this for explaining why Presage is a cybersecurity project and how it ensures user safety through monitoring, detection, incidents, and response logic.

## Attack And Logic Documents

- `attack_modules_integration_and_detection.md`
  Use this to explain each implemented attack module, the detection logic behind it, and how alerts and incidents are generated.

- `system_run_guide_and_module_workflow.md`
  Use this when you need to run the project locally and explain what each backend and frontend module does.

## Presentation And Positioning Documents

- `current_project_positioning_and_demo_guide.md`
  Use this when presenting to a moderator, reviewer, or evaluator and you need a clear explanation of where the current project stands and how to demonstrate it.

- `presentation_deck_creation_guide.md`
  Use this when preparing a slide deck for this project presentation.

## Future Product Direction

- `presage_webshield_product_roadmap.md`
  Use this to explain how Presage can evolve into a real website security product.
